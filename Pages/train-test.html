üß™ Train-Test Split Strategy

To train and evaluate our machine learning models, we used a standard practice of splitting the dataset into training and testing subsets. The training set is used to build the model, while the testing set evaluates how well the model performs on unseen data.

It is essential that the training and testing sets are disjoint ‚Äî meaning no overlap ‚Äî to avoid data leakage. This ensures our accuracy metrics truly reflect the model's generalization ability.

üîÑ Split Logic by Model

  • Na√Øve Bayes (Core & Global): Used train_test_split with an 80/20 ratio and a random_state of 42 for reproducibility. For the Core dataset, binary categorical features were encoded; for the Global dataset, numerical features were scaled and categorical ones encoded.
  • Decision Tree (Core & Global): Followed the same 80/20 disjoint split using train_test_split. Preprocessing involved encoding and selecting top clinical features.
  • Logistic Regression: Same split logic as above. The output variable was binary (PCOS Y/N), with cleaned numerical and binary inputs.
  • Multinomial Na√Øve Bayes: Applied to both datasets. For continuous features, conversion to discrete or count-like representations was considered when appropriate.
  • Ensemble Methods & SVMs: These will use the same disjoint 80/20 split, unless hyperparameter tuning or cross-validation is used, which will be clearly indicated in their respective sections.

üìå Summary

All models maintain a consistent 80/20 training-testing split. However, feature encoding and scaling may differ to suit algorithm requirements. This page will be updated if any algorithm requires a different splitting strategy.
